- Do not stress about proper formatting. Instead, get the syntax right and use the language LSP auto formatter (e.g, gofmt for golang)
- Make small surgical edits little by little
- Do not ever reset files. Do not use git to reset files
- Do not ever rewrite the whole file
- Do not be lazy because you think the file you are editing has become too unwieldy, keep working on editing until it is fixed
- Don't get frustrated and nuke an entire file by rewriting
- You should not refactor large swaths of code without a very good reason. Refactoring should be surgical.
- You should NEVER EVER WANT TO CLEAN A FILE
- Do not try to bundle edit tool calls, do s single edit tool call per fix.
- When you're about to use the edit tool, reevaluate with a read to ensure edit will succeed.
- Most command line tools are available through nix- --pure -p <tool> e.g, nix-shell --pure -p python3
- Do every edit one at a time, don't bundle edits
- Don't try to build or run the code unless you are explicitly asked to do so
